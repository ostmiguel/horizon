# Обязательства — модель «каждое обязательство = свой счёт-Пассив»

> Финальная модель (согласована 2026-07-02, заменяет single-account/ledger-first-pool).
> Причина разворота: один общий счёт «Обязательства» не может представлять N разных
> обязательств — операция `Обязательства → Расход` растит пул, но не говорит, какой
> долг вырос → дезинформация. Решение: каждое обязательство — отдельный счёт-Пассив.

## 1. Принцип

**Каждое обязательство = отдельный счёт-Пассив** (кредитка Тбанк, кредит ОТП, долг отцу…).
Баланс обязательства = **ledger этого счёта** (операции). Всегда точен, сверка не нужна.

- **Кредит** = счёт-Пассив **+ график** (loans/loan_schedule) для прогноза и амортизации.
- **Долг / кредитка** = просто счёт-Пассив (без графика). Кредитка — revolving: траты растят, платежи гасят.

Старый общий счёт «Обязательства» (пул) — **ретайрим** (деактивируем, история операций остаётся, из метрик уходит).

## 2. Операции (каждая целится в КОНКРЕТНОЕ обязательство)

| Событие | Проводка |
|---------|----------|
| Трата с кредитки | `Кредитка Тбанк → Расход` (эта кредитка растёт) |
| Погашение тела/долга/кредитки | `Карта → Кредитка Тбанк` (эта падает) |
| Проценты по кредиту | `Карта → Расход` (Кредиты-процент) |
| Страховка/комиссия | `Карта → Расход` |
| Взял кредит на карту | `Кредит ОТП → Карта` (обязательство +, кэш +) |
| Оформить обязательство | создать счёт-Пассив с начальным остатком |

Плановые платежи кредита (тело+процент по графику) подтверждаются на стр. Кредиты / в Бюджете → создают операции в **счёт этого кредита** (тело) и Расход (процент).

## 3. Метрики (агрегируются по счетам-Пассивам)

- **Чистый капитал** = активы − Σ(балансы счетов-Пассивов). Автоматически.
- **DSR** = Σ ежемесячных платежей КРЕДИТОВ (у долгов графика нет) / доход.
- **«Свободно»/прогноз** — плановые платежи кредитов (график) вычитаются; долги — по факту операций.

## 4. Данные

- **accounts**: обязательство = строка `account_type='Пассив'`. Баланс из ledger. `include_in_balance=false`.
- **loans** (только для кредитов): график/ставка/прогноз, **связь со счётом** (поле `account_name` или FK). `current_balance` больше не истина — истина в балансе счёта.
- Признак кредит/долг: кредит = у счёта есть связанный loans-график; долг = нет.

## 5. Миграция (данные владельца — по ручному пересчёту)

Начальные остатки (пересчёт владельца):
- Кредитка Тбанк 182 861 · Кредит Тбанк 286 761 · Кредит ОТП 260 641
- Кредитка Альфа 48 500 · Долг отец 50 000 · Долг Брат 30 000

Шаги:
1. Создать счёт-Пассив на каждое обязательство (opening = остаток выше).
2. Кредиты (Тбанк, ОТП) — связать существующие loans/графики со своими счетами.
3. Долги, что уже заведены как loans-записи (kind='debt') — заменить на счета (создать счёт, удалить debt-запись).
4. **Ретайрить** старый общий счёт «Обязательства» (деактивировать; из метрик уходит, история цела).

## 6. UX

- Ввод операции с типом «Обязательство» → выбор КОНКРЕТНОГО счёта-Пассива (не общий пул).
- Страница «Кредиты» = список счетов-обязательств: кредиты с графиком (богато), долги/кредитки — остаток + гашение.
- Сверка (Фаза 3) — **убирается** (каждый счёт самосогласован).

## 7. Фазы (переработка)

1. **Данные:** `loans.account_name` (связь кредита со счётом); ничего не ломаем.
2. **Миграция владельца:** создать 6 счетов-Пассивов, связать кредиты, конвертировать долги, ретайрить пул. С проверкой цифр.
3. **Операции:** тип «Обязательство» → выбор конкретного счёта; материализация графика кредита → счёт кредита (не пул).
4. **Страница «Кредиты»:** рендер из счетов-обязательств; убрать сверку.
5. **«Информация»/онбординг** (обязательно) — новая модель, как заводить, как вносить.
